# 超级学长回访自动填写脚本：使用教程

## 这个脚本会做什么

打开“学管沟通回访表”后，页面右下角会出现一个小面板。点击“开始处理当前页”后，它会自动逐条处理当前页的“待处理”记录：

- 日期：在“今天往前 5 天内”随机选择一个日期
- 开始时间：10:00 到 20:00 之间随机
- 结束时间：比开始时间晚 5 到 15 分钟
- 所有文字框：填写“学习情况沟通”
- 反馈状态：选择“正常”
- 默认会点击“确定”自动提交

## 第一次安装

1. 用 Chrome 打开扩展商店，安装 Tampermonkey（油猴）。
2. 安装后，点击浏览器右上角 Tampermonkey 图标。
3. 选择“添加新脚本”。
4. 删除编辑器里原来的内容。
5. 打开这个文件并复制全部内容：

   `C:\Users\Rebecca\Documents\New project\超级学长回访自动填写.user.js`

6. 粘贴到 Tampermonkey 编辑器里。
7. 按 `Ctrl + S` 保存。
8. 回到 CRM 的“学管沟通回访表”页面，刷新页面。
9. 右下角出现“学管回访自动填写”面板，就代表安装成功。

## 日常使用

1. 打开 CRM：学管沟通回访表。
2. 筛选好你要处理的记录。
3. 建议先每页 10 条。
4. 点击右下角“开始处理当前页”。
5. 等脚本自动完成。
6. 如果还有下一页，翻页后再点一次“开始处理当前页”。

## 重要设置

打开脚本顶部，可以看到：

```js
const CONFIG = {
  maxPerRun: 10,
  autoSubmit: true,
  randomDateBackDays: 5,
  startHour: 10,
  endHour: 20,
  minDurationMinutes: 5,
  maxDurationMinutes: 15,
  textValue: '学习情况沟通',
  waitAfterSubmitMs: 1800
};
```

常用修改：

### 1. 先测试，不自动提交

把：

```js
autoSubmit: true,
```

改成：

```js
autoSubmit: false,
```

这样脚本只填写，不点“确定”。确认没问题后再改回 `true`。

### 2. 每次只处理 1 条

把：

```js
maxPerRun: 10,
```

改成：

```js
maxPerRun: 1,
```

### 3. 日期范围

现在是当天往前 5 天内随机：

```js
randomDateBackDays: 5,
```

如果以后想改成当天往前 3 天，就改成：

```js
randomDateBackDays: 3,
```

## 注意事项

- 使用前请确认这些记录确实需要填写并提交。
- 如果页面结构改版，脚本可能需要调整。
- 如果网络慢，可以把 `waitAfterSubmitMs` 改大，例如 `2500` 或 `3000`。
- 如果脚本弹窗提示停止，请先看右下角日志，通常是页面没加载完、弹窗没打开、或字段没写入。
